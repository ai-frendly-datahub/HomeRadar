# HomeRadar 로드맵 (Roadmap)

## Phase 0: 프로젝트 초기화 (✅ 완료)
부동산 시장 데이터 수집을 위한 기본 아키텍처와 스켈레톤 코드를 구축했습니다.
- [x] 프로젝트 구조 초기화 및 Radar 시리즈 공통 아키텍처 적용
- [x] 핵심 문서 작성 (ARCHITECTURE.md, PRD.md, BRANDING.md)
- [x] BaseCollector 및 RawItem 모델 정의
- [x] 데이터 소스 설정 파일 (`config/sources.yaml`) 작성 (15개 소스 정의)
- [x] TDD 기반 테스트 환경 구축 (`pytest` 연동)

## Phase 1: 기본 수집 파이프라인 (📋 계획/진행 중)
실제 부동산 데이터를 수집하고 저장하는 핵심 파이프라인을 완성합니다.
- [x] **국토교통부(MOLIT) API 수집기**: 아파트 매매/전월세 실거래가 수집 구현
- [x] **RSS 뉴스 수집기**: 부동산 전문 뉴스(뉴스랜드 등) 수집 구현
- [x] **그래프 저장소 (DuckDB)**: `urls`, `entities`, `transactions` 테이블 및 관계 저장
- [x] **엔터티 추출기**: 지역명, 단지명, 브랜드, 사업명 키워드 추출 (364개 엔터티)
- [ ] **HTML 리포터**: 일일 시장 신호를 시각화하는 HTML 리포트 생성기 구현
- [ ] **GitHub Actions**: 일일 자동 수집 및 GitHub Pages 배포 워크플로 설정
- [ ] **메인 스케줄러**: `main.py`를 통한 통합 실행 및 스케줄링 로직 완성

## Phase 2: 분석 고도화 및 알림 (예상 2주)
수집된 데이터를 분석하여 유의미한 시장 신호를 추출하고 사용자에게 알립니다.
- [ ] **가격 변동 분석**: 단지별 실거래가 추이 및 전세-매매 갭 변화 분석 로직 추가
- [ ] **스코어링 시스템**: 급등/급락, 거래량 변화, 호재 키워드 기반 "주목 단지" 점수화
- [ ] **알림 시스템**: Telegram 봇 및 Email을 통한 주요 시장 신호 즉시 알림
- [ ] **자연어 쿼리 (NLQ)**: `nl_query.py` 고도화를 통한 부동산 데이터 검색 강화
- [ ] **MCP 서버 확장**: Claude Desktop 연동 툴(get_deals, search_apt) 추가

## Phase 3: 서비스 안정화 및 확장 (예상 4주)
데이터 소스를 확장하고 시스템의 안정성을 확보합니다.
- [ ] **추가 데이터 소스**: 네이버 부동산, 청약홈 분양 정보 등 수집 대상 확대
- [ ] **데이터 품질 검증**: 수집 데이터 무결성 체크 및 이상치 탐지 로직 강화
- [ ] **시각화 대시보드**: Chart.js 기반의 지역별 가격 변동 그래프 리포트 추가
- [ ] **운영 모니터링**: 수집 성공률 추적 및 에러 리트라이 로직 안정화

## KPI 목표 (Phase별)

### Phase 1 (기본 파이프라인)
- 일일 수집 성공률 ≥ 95%
- 수집 데이터 건수 ≥ 500건/일
- 테스트 커버리지 ≥ 80%

### Phase 2 (분석 및 알림)
- 시장 신호 감지 정확도 ≥ 85%
- 알림 발송 성공률 ≥ 98%
- 리포트 생성 및 배포 자동화 성공률 100%

### Phase 3 (안정화 및 확장)
- 데이터 소스 다양성 (T1/T2/T3 균형 확보)
- 사용자 쿼리 응답 시간 ≤ 2초
- 시스템 가동률(Uptime) ≥ 99.9%
